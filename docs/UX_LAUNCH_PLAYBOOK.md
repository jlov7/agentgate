# UX Launch Playbook

## Pre-Launch

1. Freeze copy on primary journey pages.
2. Run full verification (`make verify`, `scripts/doctor.sh`).
3. Validate onboarding, sandbox, and journey transitions manually.
4. Confirm UX tracker status and accepted residual gaps.

## Launch Day

1. Announce role-based start page and journey map updates.
2. Share quick-start path for trial users and operators.
3. Monitor workflow errors and support request categories hourly.

## Post-Launch (First 72h)

1. Review funnel and friction metrics.
2. Triage top user confusion points.
3. Ship microcopy/navigation hotfixes quickly.
